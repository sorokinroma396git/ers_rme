## 1. Как устроен процесс

Мини-приложение «Сервисы региона» — это витрина всех цифровых сервисов вашего региона внутри мессенджера Max. Жители смогут найти нужный сервис, перейти в чат-бот, на сайт, в мини-приложение, канал или заполнить форму.

Чтобы запустить приложение для своего региона, нужно заполнить файл настроек и подготовить картинки. Редактирование JSON не требует навыков программирования. Исходный код проекта передаётся региону целиком — вы можете самостоятельно проверить конфигурацию и собрать готовый сайт (см. [руководство по сборке](region-build-guide.md)).

**Порядок работы:**

1. **Получаете проект** — репозиторий или архив с исходным кодом, этой инструкцией и примером файла настроек
2. **Устанавливаете окружение** — Node.js и Yarn (версии указаны в [руководстве по сборке](region-build-guide.md))
3. **Создаете и заполняете `static/region.json`** — перечисляете сервисы, каналы, категории, поддержку, при необходимости — блок аналитики
4. **Кладёте картинки** в папки `static/static/categories/`, `static/static/channels/`, `static/static/services/` (см. [раздел 5](#5-работа-с-картинками)); в JSON указываете **только имена файлов**
5. **Запускаете проверку** — в корне проекта: `yarn validate`. Если есть ошибки, их текст подскажет, что исправить (расшифровка — [в разделе 7](#7-ошибки-валидации-и-как-их-исправить))
6. **Собираете приложение** — `yarn build`; результат — папка `public/` (подробности в [руководстве по сборке](region-build-guide.md))
7. **Публикуете** — размещаете содержимое `public/` на своём хостинге или в инфраструктуре, принятой у вас для мини-приложения

> **Название региона** задаётся отдельно в настройках бота и **не входит** в файл настроек.

## 2. Где лежат настройки и картинки в проекте

В репозитории файлы раскладываются так:

```
<корень проекта>/
├── static/
│   ├── region.json          ← файл настроек
│   ├── channels-and-services.zip  ← архив с примерами картинок каналов и сервисов
│   └── static/
│       ├── categories/      ← иконки категорий для главной
│       │   ├── housing.png
│       │   └── …
│       ├── channels/        ← аватары для карусели каналов (создайте и наполните)
│       └── services/        ← иконки для всех сервисов, которые не выводятся в карусели каналов вверху главной страницы (создайте и наполните)
├── docs/                    ← документация, в т.ч. эта инструкция
└── …                        ← исходный код приложения
```

> **Важно:** файл `region.json` отсутствует в репозитории.
>
> В архиве `static/channels-and-services.zip` лежат примеры картинок для каналов и сервисов — можно использовать как образец.

- **region.json** — главный файл, в котором описано всё содержимое приложения
- **categories/** — картинки для категорий
- **channels/** — аватары для карусели каналов на главном экране (губернатор, ведомства и т.д.)
- **services/** — иконки для всех сервисов, которые не выводятся в карусели каналов вверху главной страницы

## 3. Правила редактирования файла настроек

Файл `region.json` имеет специальный формат. Вот несколько простых правил, которые нужно соблюдать при редактировании:

> **Рекомендация:** открывайте файл в любом текстовом редакторе (Блокнот, TextEdit и т.д.). Для проверки правильности формата используйте сайт [jsonlint.com](https://jsonlint.com/) — вставьте содержимое файла, и он покажет, есть ли ошибки и где именно. **Не используйте** Word, Google Docs или Pages — они могут испортить формат файла.

### Основные правила

1. **Весь текст пишется в двойных кавычках**: `"Название сервиса"`. Одинарные кавычки (`'текст'`) не подойдут
2. **Значения да/нет пишутся без кавычек**: `true` (да) и `false` (нет)
3. **Списки оформляются в квадратных скобках** через запятую:

```
 "tags": ["тег1", "тег2", "тег3"]
```

4. **После последнего элемента в списке запятая НЕ ставится**:

```
 Правильно:  ["тег1", "тег2", "тег3"]
 Неправильно: ["тег1", "тег2", "тег3",]
```

5. **Каждый блок (сервис, канал и т.д.) оформляется в фигурных скобках**, элементы блока разделяются запятыми:

```
 {
   "id": "my-service",
   "name": "Мой сервис"
 }
```

6. **Между блоками в списке ставится запятая**, после последнего — нет:

```
 "services": [
   { ... },
   { ... },
   { ... }
 ]
```

## 4. Заполнение файла настроек

Файл настроек состоит из 5 разделов. Все разделы обязательны (даже если оставляете некоторые пустыми).

```
{
  "channels": [ ... ],
  "categories": [ ... ],
  "services": [ ... ],
  "support": { ... },
  "analytics": { ... }
}
```

---

### 4.1. Официальные каналы (channels)

Каналы — это официальные аккаунты вашего региона (губернатор, ведомства, новостные каналы). Они отображаются на главном экране приложения в горизонтальной ленте с аватарами.

**Если каналов нет** — оставьте пустой список: `"channels": []`. Лента просто не будет отображаться.

**Пример одного канала:**

```
{
  "id": "channel-governor",
  "name": "Иван Иванов",
  "description": "Официальный канал губернатора Новгородской области.",
  "icon": "governor.png",
  "type": "chatbot",
  "owner": "Правительство Новгородской области",
  "link": "https://max.ru/user/governor_novgorod",
  "tags": ["губернатор", "новости"],
  "category_ids": ["housing", "social"]
}
```

**Описание полей:**

| Поле           | Что писать                                                                                                                                                            | Обязательно? |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------ |
| `id`           | Уникальный код канала. Придумайте короткое название латиницей: `"channel-governor"`, `"channel-mfc"`. Не должен повторяться ни у одного сервиса или канала            | Да           |
| `name`         | Название, которое увидят пользователи: `"Иван Иванов"`, `"МФЦ области"`                                                                                               | Да           |
| `description`  | Подробное описание канала. Будет показано на странице канала                                                                                                          | Да           |
| `icon`         | Имя файла картинки в папке `channels/` — см. [раздел 5](#5-работа-с-картинками). Можно указать полный URL. Если не указать — покажется заглушка | Нет          |
| `type`         | Тип канала — одно из 5 значений (см. [таблицу типов](#таблица-типов) ниже)                                                                                            | Да           |
| `owner`        | Организация-владелец: `"Правительство области"`                                                                                                                       | Да           |
| `link`         | Ссылка на канал. Должна начинаться с `https://` или `http://`                                                                                                         | Да           |
| `tags`         | Слова для поиска. Через них пользователи смогут найти канал                                                                                                           | Нет          |
| `category_ids` | Коды категорий, к которым относится канал (см. [раздел 4.2](#42-категории-сервисов-categories)). Если указать — канал также появится в списке сервисов этих категорий | Нет          |
| `popular`      | `true` или `false` — популярный канал (выше в результатах поиска)                                                                                                     | Нет          |
| `federal`      | `true` или `false` — федеральный канал (выше в результатах поиска, но ниже популярных)                                                                                | Нет          |

### 4.2. Категории сервисов (categories)

Категории — это разделы, по которым группируются сервисы: «Дом и ЖКХ», «Здоровье», «Транспорт» и т.д.

В файле уже предзаполнены 10 стандартных категорий. Вы можете:

- Оставить как есть
- Удалить ненужные
- Добавить свои
- Поменять порядок

**Стандартные категории:**

| Код         | Название     | Показывать на главном экране? |
| ----------- | ------------ | ----------------------------- |
| `housing`   | Дом и ЖКХ    | Да                            |
| `family`    | Семья и дети | Да                            |
| `transport` | Транспорт    | Да                            |
| `health`    | Здоровье     | Да                            |
| `culture`   | Культура     | Да                            |
| `education` | Образование  | Нет                           |
| `social`    | Соцподдержка | Нет                           |
| `ecology`   | Экология     | Нет                           |
| `incidents` | Происшествия | Нет                           |
| `business`  | Бизнес       | Нет                           |

**Пример категории:**

```
{
  "id": "housing",
  "name": "Дом и ЖКХ",
  "icon": "housing.png",
  "show_on_main": true
}
```

**Описание полей:**

| Поле           | Что писать                                                                                                                                            | Обязательно?               |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------- |
| `id`           | Уникальный код категории латиницей: `"housing"`, `"health"`                                                                                           | Да                         |
| `name`         | Название категории: `"Дом и ЖКХ"`, `"Здоровье"`                                                                                                       | Да                         |
| `icon`         | Имя файла в папке `categories/` (или URL). **Обязательно**, если категория на главном экране                                                          | Только для главного экрана |
| `show_on_main` | `true` — показывать на главном экране в виде карточки с картинкой. `false` (или не указывать) — категория будет доступна только через фильтры и поиск | Нет                        |

**Важные ограничения:**

- На главном экране должно быть **от 2 до 5 категорий** с `show_on_main: true`
- Для всех категорий с `show_on_main: true` **картинка обязательна**
- Категория, в которой нет ни одного сервиса, не будет показана пользователям

### 4.3. Сервисы (services)

Сервисы — основное содержимое приложения. Это чат-боты, сайты, мини-приложения, формы и каналы вашего региона.

**Пример сервиса:**

```
{
  "id": "zhkh-emergency",
  "name": "Аварийная служба ЖКХ",
  "description": "Бот помогает быстро сообщить о коммунальной аварии и получить контакты ответственных служб.",
  "icon": "zhkh-emergency.png",
  "type": "chatbot",
  "owner": "Администрация области",
  "link": "https://max.ru/bot/zhkh_novgorod",
  "tags": ["коммуналка", "авария", "отопление"],
  "category_ids": ["housing"],
  "popular": true,
  "federal": false
}
```

**Описание полей:**

| Поле           | Что писать                                                                                                                                                                   | Обязательно? |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------ |
| `id`           | Уникальный код сервиса латиницей: `"zhkh-emergency"`, `"gosuslugi"`. Не должен повторяться ни у одного сервиса или канала                                                    | Да           |
| `name`         | Название сервиса: `"Аварийная служба ЖКХ"`                                                                                                                                   | Да           |
| `description`  | Подробное описание — что делает сервис, чем полезен                                                                                                                          | Да           |
| `icon`         | Имя файла в папке `services/` или URL. Если не указать — покажется заглушка                                                                                                  | Нет          |
| `type`         | Тип сервиса — одно из 5 значений (см. [таблицу типов](#таблица-типов) ниже)                                                                                                  | Да           |
| `owner`        | Организация-владелец: `"Администрация области"`                                                                                                                              | Да           |
| `link`         | Ссылка. Должна начинаться с `https://` или `http://`                                                                                                                         | Да           |
| `tags`         | Слова для поиска. Рекомендуется 3–7 тегов. Пользователь найдёт сервис, введя любой из этих тегов                                                                             | Нет          |
| `category_ids` | Коды категорий, к которым относится сервис. **Минимум одна категория.** Можно указать несколько — сервис появится во всех. Коды должны совпадать с `id` из раздела категорий | Да           |
| `popular`      | `true` — популярный сервис (будет выше в результатах поиска)                                                                                                                 | Нет          |
| `federal`      | `true` — федеральный сервис (выше в результатах поиска, но ниже популярных)                                                                                                  | Нет          |

---

### Таблица типов

Тип определяет, что произойдёт, когда пользователь нажмёт кнопку «Открыть» на странице сервиса.

| Что писать в поле `type` | Что это               | Куда ведёт ссылка                 | Пример ссылки                           |
| ------------------------ | --------------------- | --------------------------------- | --------------------------------------- |
| `"chatbot"`              | Чат-бот в мессенджере | Бот или пользователь внутри Max   | `https://max.ru/bot/zhkh_kaluga`        |
| `"site"`                 | Внешний сайт          | Любой сайт в интернете            | `https://edu.admoblkaluga.ru`           |
| `"miniApp"`              | Мини-приложение       | Другое мини-приложение внутри Max | `https://max.ru/gosuslugi_bot?startapp` |
| `"form"`                 | Опрос или анкета      | Яндекс Формы, Google Forms и т.п. | `https://forms.yandex.ru/cloud/...`     |
| `"channel"`              | Канал в мессенджере   | Канал внутри Max                  | `https://max.ru/channel/kaluga_official` |

> **Обратите внимание на написание:** тип `"miniApp"` пишется именно так — с большой буквой A в середине. Тип `"chatbot"` — слитно, маленькими буквами. Тип `"channel"` — слитно, маленькими буквами.
>
> **Мини-приложения:** ссылка должна содержать `?startapp` на конце — это позволяет автоматически запустить мини-приложение при переходе по ссылке.

### 4.4. Поддержка (support)

Блок поддержки отображается внизу главного экрана. Здесь можно указать до 3 элементов: чат-бот поддержки, чат с оператором и техническую поддержку.

**Пример:**

```
"support": {
  "chatbot": {
    "name": "Чат-бот поддержки",
    "description": "Автоматическое решение вопросов круглосуточно",
    "link": "https://max.ru/bot/support_novgorod"
  },
  "operator": {
    "name": "Чат с оператором",
    "description": "Задайте вопрос и получите ответ в течение 24 часов",
    "link": "https://max.ru/chat/operator_novgorod"
  },
  "techSupport": null
}
```

**Три элемента поддержки:**

| Название в файле | Что это                |
| ---------------- | ---------------------- |
| `chatbot`        | Чат-бот поддержки      |
| `operator`       | Чат с живым оператором |
| `techSupport`    | Техническая поддержка  |

Каждый элемент, если он заполнен, должен содержать:

| Поле          | Что писать                                                          |
| ------------- | ------------------------------------------------------------------- |
| `name`        | Заголовок: `"Чат-бот поддержки"`                                    |
| `description` | Краткое описание: `"Автоматическое решение вопросов круглосуточно"` |
| `link`        | Ссылка для перехода (начинается с `https://` или `http://`)         |

**Если какой-то элемент не нужен** — напишите `null` (без кавычек):

```
"techSupport": null
```

Если все три элемента указаны как `null` — блок поддержки не будет показан в приложении.

### 4.5. Аналитика (analytics)

Приложение может собирать статистику использования: сколько раз открывали приложение, какие сервисы искали, по каким категориям и каналам переходили. Для этого поддерживаются два счётчика аналитики:

- **MyTracker** — см. [tracker.my.com](https://tracker.my.com)
- **Яндекс.Метрика** — см. [metrika.yandex.ru](https://metrika.yandex.ru)

Можно подключить один из них, оба сразу или ни одного.

**Если вы хотите сами отслеживать статистику:**

1. **MyTracker** — зарегистрируйтесь на [tracker.my.com](https://tracker.my.com), создайте счётчик для сайта и скопируйте его числовой идентификатор (Tracker ID)
2. **Яндекс.Метрика** — зарегистрируйтесь на [metrika.yandex.ru](https://metrika.yandex.ru), создайте счётчик и скопируйте его номер

Впишите идентификатор в файл настроек и измените `false` на `true`:

```
"analytics": {
  "mytracker": {
    "enabled": true,
    "trackerId": "1234567"
  },
  "yandexMetrika": {
    "enabled": true,
    "counterId": "98765432"
  }
}
```

**Если вы не планируете отслеживать статистику самостоятельно** — оставьте раздел без изменений.

**Значения по умолчанию (аналитика отключена):**

```
"analytics": {
  "mytracker": {
    "enabled": false,
    "trackerId": ""
  },
  "yandexMetrika": {
    "enabled": false,
    "counterId": ""
  }
}
```

**Какие данные собираются:**

Приложение автоматически отправляет в подключённые счётчики информацию о действиях пользователей:

- Открытие приложения
- Поисковые запросы и пустые результаты поиска
- Переходы в сервисы и нажатия кнопки «Открыть»
- Клики по категориям на главном экране
- Клики по каналам в ленте
- Клики по элементам блока поддержки

Персональные данные пользователей не собираются и не передаются.

## 5. Работа с картинками

### Где какие картинки нужны

В проекте это каталоги **внутри** `static/static/`:

| Для чего                               | Куда класть в репозитории   | Рекомендуемый размер | Форматы                        |
| -------------------------------------- | --------------------------- | -------------------- | ------------------------------ |
| Аватар канала в карусели на главной     | `static/static/channels/`   | 56 × 56 пикселей     | PNG, JPG                       |
| Иконка категории (Дом и ЖКХ, Здоровье) | `static/static/categories/` | 48 × 48 пикселей     | PNG (лучше с прозрачным фоном) |
| Иконка сервиса (любого типа)            | `static/static/services/`   | 80 × 80 пикселей     | PNG, JPG                       |

> **Обратите внимание:** папка `channels/` используется **только** для аватаров в карусели каналов на главном экране (раздел `channels` в JSON). Иконки всех сервисов — включая сервисы с типом `"channel"` — хранятся в папке `services/`.

### Как добавить свою картинку

1. Подготовьте изображение нужного размера
2. Назовите файл **латиницей**, без пробелов. Можно использовать буквы, цифры и дефис

- Правильно: `governor.png`, `zhkh-emergency.png`, `mfc-novgorod.jpg`
- Неправильно: `губернатор.png`, `мой сервис.png`, `фото (1).png`

3. Положите файл в нужную подпапку (`channels`, `categories` или `services` внутри `static/static/`)
4. В `region.json` в поле `icon` укажите **только имя файла**, например: `"icon": "governor.png"`

Запись формата (`"categories/housing.png"`) приложение по-прежнему поймёт, но для конфигов достаточно имени файла.

### Как заменить стандартную картинку

Положите в `static/static/categories/` новый файл с тем же именем, что указано в JSON.

### Можно ли использовать картинку из интернета?

Да. Вместо пути к файлу укажите полную ссылку:

```
"icon": "https://cdn.example.com/icons/my-icon.png"
```

В этом случае класть файл в архив не нужно.

### Поддерживаемые форматы файлов

`.png`, `.jpg`, `.jpeg`, `.svg`, `.webp`

## 6. Контрольный чек-лист перед отправкой

Перед сборкой и публикацией убедитесь:

- Файл `static/region.json` открывается без ошибок (нет лишних запятых, все кавычки закрыты)
- У каждого сервиса и канала есть уникальный `id` (латиницей)
- У каждого сервиса заполнены: `id`, `name`, `description`, `type`, `owner`, `link`, `category_ids`
- У каждого канала заполнены: `id`, `name`, `description`, `type`, `owner`, `link`
- Тип (`type`) — одно из пяти: `chatbot`, `site`, `miniApp`, `form`, `channel`
- Все ссылки начинаются с `https://` или `http://`
- Категорий с `show_on_main: true` — от 2 до 5 штук
- У каждой категории с `show_on_main: true` есть картинка
- Каждый сервис привязан хотя бы к одной категории
- Коды категорий в `category_ids` совпадают с `id` в разделе категорий
- Все указанные картинки есть на диске в `static/static/` (в правильных подпапках)
- Имена файлов картинок — латиницей, без пробелов

## 7. Ошибки валидации и как их исправить

Командой `yarn validate` в корне проекта запускается автоматическая проверка конфигурации. Если что-то заполнено неправильно, в терминале появится список ошибок. Ниже — все возможные сообщения с объяснениями, что пошло не так и как исправить.

> **Ошибки** (ERRORS) блокируют запуск приложения — их нужно обязательно исправить.
> **Предупреждения** (WARNINGS) не мешают запуску, но сообщают о потенциальных проблемах.

---

### Общие ошибки

#### «Файл конфигурации не найден»

**Что случилось:** Файл `static/region.json` не найден.

**Что делать:** Файл `region.json` не входит в репозиторий — его нужно создать внутри папки `/static`.

---

#### «Ошибка парсинга JSON»

**Что случилось:** Файл настроек повреждён — скорее всего, допущена опечатка в формате.

**Что делать:** Проверьте файл на типичные ошибки:

- Лишняя запятая после последнего элемента в списке
- Незакрытая кавычка `"`
- Пропущена запятая между элементами
- Используются одинарные кавычки `'` вместо двойных `"`

> **Совет:** вставьте содержимое файла на сайт [jsonlint.com](https://jsonlint.com/) — он покажет, где именно ошибка.

---

#### «Отсутствует обязательное корневое поле»

**Что случилось:** В файле не хватает одного из 5 обязательных разделов.

**Что делать:** Убедитесь, что в файле есть все 5 разделов: `channels`, `categories`, `services`, `support`, `analytics`. Даже если каналов нет — раздел должен быть с пустым списком: `"channels": []`.

---

### Ошибки в каналах и сервисах

#### «Дубликат id»

**Пример:** `Дубликат id: "my-service"`

**Что случилось:** Два или более сервиса/канала имеют одинаковый код (`id`).

**Что делать:** Найдите повторяющийся `id` и измените его. Каждый сервис и канал должен иметь свой уникальный код. Коды не могут повторяться даже между каналами и сервисами.

---

#### «Поле "id" сервиса/канала должно быть непустой строкой»

**Что случилось:** У одного из сервисов или каналов не заполнен или отсутствует код `id`.

**Что делать:** Впишите уникальный код латиницей: `"id": "my-service"`. Можно использовать латинские буквы, цифры и дефис.

---

#### «Поле "name" должно быть непустой строкой»

**Что случилось:** У сервиса или канала не заполнено название.

**Что делать:** Впишите название: `"name": "Аварийная служба ЖКХ"`.

---

#### «Поле "description" должно быть непустой строкой»

**Что случилось:** У сервиса или канала не заполнено описание.

**Что делать:** Впишите описание: `"description": "Бот помогает сообщить о коммунальной аварии."`.

---

#### «Поле "owner" должно быть непустой строкой»

**Что случилось:** У сервиса или канала не указана организация-владелец.

**Что делать:** Впишите название организации: `"owner": "Администрация области"`.

---

#### «Недопустимое значение type»

**Пример:** `Недопустимое значение type: "bot"`

**Что случилось:** В поле `type` указано неизвестное значение.

**Что делать:** Используйте строго одно из пяти значений:

- `"chatbot"` — для чат-бота
- `"site"` — для сайта
- `"miniApp"` — для мини-приложения (обратите внимание: буква **A** — большая)
- `"form"` — для опроса/анкеты
- `"channel"` — для канала в мессенджере

---

#### «Некорректный URL в поле "link"»

**Пример:** `Некорректный URL в поле "link": "max.ru/bot/test"`

**Что случилось:** Ссылка указана в неправильном формате.

**Что делать:** Убедитесь, что ссылка начинается с `https://` или `http://`:

- Правильно: `"link": "https://max.ru/bot/test"`
- Неправильно: `"link": "max.ru/bot/test"`

---

#### «Поле "category_ids" обязательно и должно содержать минимум 1 категорию»

**Что случилось:** У сервиса не указано, к какой категории он относится.

**Что делать:** Добавьте хотя бы одну категорию: `"category_ids": ["housing"]`. Код категории должен совпадать с `id` из раздела категорий.

---

#### «Ссылка на несуществующую категорию»

**Пример:** `Ссылка на несуществующую категорию: "zdorovie"`

**Что случилось:** В `category_ids` указан код категории, которого нет в разделе категорий.

**Что делать:** Проверьте, что код в `category_ids` точно совпадает с `id` в разделе `categories`. Например, если в категориях `id` — `"health"`, то и в сервисе должно быть `"health"`, а не `"zdorovie"` или `"Health"`.

---

#### «Поле "category_ids" должно быть массивом строк» / «Все элементы category_ids должны быть строками»

**Что случилось:** Список категорий записан в неправильном формате.

**Что делать:** Категории записываются как список кодов в кавычках и квадратных скобках:

- Правильно: `"category_ids": ["housing", "social"]`
- Неправильно: `"category_ids": [housing, social]`
- Неправильно: `"category_ids": "housing"` (это одна строка, а нужен список)

---

#### «Поле "tags" должно быть массивом строк»

**Что случилось:** Теги указаны в неправильном формате.

**Что делать:** Теги записываются как список слов в кавычках:

- Правильно: `"tags": ["вода", "авария", "ЖКХ"]`
- Неправильно: `"tags": "вода, авария, ЖКХ"` (это одна строка, а нужен список)

Если тегов нет — просто удалите всю строку с `"tags"` целиком.

---

#### «Поле "popular"/"federal" должно быть boolean»

**Что случилось:** В поле `popular` или `federal` указано что-то кроме `true` или `false`.

**Что делать:** Используйте только `true` или `false` **без кавычек**:

- Правильно: `"popular": true`
- Неправильно: `"popular": "true"` (тут true в кавычках — так нельзя)
- Неправильно: `"popular": "да"`

Если поле не нужно — удалите его целиком (по умолчанию считается `false`).

---

### Ошибки в категориях

#### «Дубликат id категории»

**Пример:** `Дубликат id категории: "health"`

**Что случилось:** Две категории имеют одинаковый код.

**Что делать:** Убедитесь, что каждая категория имеет уникальный `id`.

---

#### «Поле "id"/"name" категории должно быть непустой строкой»

**Что случилось:** У категории не заполнен код или название.

**Что делать:** Заполните оба поля: `"id": "health"`, `"name": "Здоровье"`.

---

#### «Категорий с show_on_main: true менее 2 или более 5»

**Пример:** `Найдено: 1. Допустимо: от 2 до 5`

**Что случилось:** На главный экран вынесено слишком мало или слишком много категорий.

**Что делать:** Убедитесь, что `show_on_main: true` стоит **у 2–5 категорий**. Остальные категории оставьте без этого признака — они будут доступны через фильтры и поиск.

---

#### «Категория с show_on_main: true не имеет иконки»

**Пример:** `Категория "Здоровье" с show_on_main: true не имеет иконки`

**Что случилось:** Категория выведена на главный экран, но для неё не указана картинка.

**Что делать:** Либо добавьте картинку: положите файл в `static/static/categories/` и укажите `"icon": "health.png"` (или другое имя), либо уберите категорию с главного экрана: удалите строку `"show_on_main": true`.

---

### Ошибки в картинках

#### «Файл не найден»

**Пример:** `Файл не найден: "services/my-icon.png"`

**Что случилось:** В файле настроек указан путь к картинке, но в архиве этого файла нет.

**Что делать:** Проверьте:

1. Файл лежит в `static/static/channels/`, `static/static/categories/` или `static/static/services/`
2. В поле `icon` указано имя файла без папки (например `icon.png`) и оно совпадает с именем на диске (с учётом регистра: `Icon.png` и `icon.png` — разные файлы)

---

#### «Неподдерживаемый формат изображения»

**Пример:** `Неподдерживаемый формат изображения: ".bmp"`

**Что случилось:** Картинка в неподдерживаемом формате.

**Что делать:** Используйте один из допустимых форматов: **PNG**, **JPG**, **JPEG**, **SVG**, **WEBP**. Пересохраните картинку в нужном формате.

---

#### «Некорректный URL» (для внешних картинок)

**Что случилось:** Если вы указали ссылку на картинку из интернета, она записана в неправильном формате.

**Что делать:** Ссылка должна начинаться с `https://` или `http://`:

- Правильно: `"icon": "https://cdn.example.com/icon.png"`
- Неправильно: `"icon": "cdn.example.com/icon.png"`

---

### Ошибки в блоке поддержки

#### «Поле "name"/"description" обязательно для заполненного блока поддержки»

**Что случилось:** Блок поддержки частично заполнен — не хватает названия или описания.

**Что делать:** Если блок заполняется — укажите все три поля (`name`, `description`, `link`). Если блок не нужен — укажите `null`:

```
"chatbot": null
```

---

#### «Некорректный URL в поле "link"» (поддержка)

**Что случилось:** В блоке поддержки указана неправильная ссылка.

**Что делать:** Убедитесь, что ссылка начинается с `https://` или `http://`.

---

### Ошибки в аналитике

#### «trackerId / counterId не заполнен при включённом счётчике»

**Пример:** `При mytracker.enabled === true поле "trackerId" должно быть непустой строкой`

**Что случилось:** Счётчик аналитики включён, но не указан его идентификатор.

**Что делать:** Либо впишите идентификатор, либо отключите счётчик — измените `true` на `false`:

```
"mytracker": {
  "enabled": false,
  "trackerId": ""
}
```

---

### Предупреждения (не мешают запуску)

Эти сообщения не блокируют публикацию, но обращают внимание на потенциальные проблемы.

#### «Массив каналов пуст — карусель не будет отображаться»

Раздел каналов пуст. Если каналы не нужны — это нормально, лента каналов просто не покажется.

#### «Категория не содержит привязанных сервисов — не будет отображаться»

Категория есть в списке, но ни один сервис к ней не привязан. Категория не будет видна пользователям. Чтобы она появилась — добавьте сервис и укажите эту категорию в его `category_ids`.

#### «Ни один элемент поддержки не заполнен — блок не будет отображаться»

Все три элемента поддержки указаны как `null`. Блок поддержки не будет показан. Если поддержка не нужна — это нормально.
