# Сборка и проверка мини-приложения «Сервисы региона»

Кратко: что установить и какие команды выполнить в папке с проектом.

---

## Требования

- **Node.js** — актуальная LTS-версия (минимальная версия 20.x)
- **Yarn** версии 1.x

Проверка:

```bash
node -v
yarn -v
```

---

## Установка зависимостей

В корне репозитория:

```bash
yarn install
```

---

## Пример работы приложения

Чтобы быстро запустить приложение с тестовыми данными:

1. **Переименуйте** файл `static/region.example.json` в `static/region.json`
2. **Разархивируйте** `static/channels-and-services.zip` — в нём уже подготовлены тестовые картинки. Поместите распакованные папки `channels/` и `services/` внутрь папки `static/static/`.
3. **Установите зависимости:**
   ```bash
   yarn install
   ```
4. **Запустите проект локально:**
   ```bash
   yarn dev
   ```

Приложение откроется на `http://localhost:8080`.

---

## Где лежат настройки и картинки

| Назначение       | Папка                        |
| ---------------- | ---------------------------- |
| Файл настроек    | `static/region.json`         |
| Иконки категорий | `static/static/categories/`  |
| Аватары каналов (карусель на главной странице) | `static/static/channels/`    |
| Иконки сервисов | `static/static/services/`    |

> **Первый шаг:** создайте файл `static/region.json` и заполните его данными своего региона.
>
> 1. Изучите структуру файла настроек в `docs/region-instruction.md`.
> 2. Создайте `static/region.json` и внесите данные своего региона.
>
> Примеры картинок для каналов и сервисов находятся в архиве `static/channels-and-regions.zip`. Папки `static/static/channels/` и `static/static/services/` нужно создать и наполнить картинками вашего региона.

В полях `icon` в JSON указывайте **имя файла** (например `housing.png`). Можно указать полный адрес картинки в интернете (`https://…`).

---

## Проверка настроек

```bash
yarn validate
```

Если в конце написано, что проверка **пройдена** и **ошибок нет** — можно собирать проект.

Если есть **ошибки (ERRORS)** — их нужно исправить; сборка без исправлений не пройдёт. Что значит каждое сообщение — в `docs/region-instruction.md`, раздел «Ошибки валидации».

**Предупреждения (WARNINGS)** сборку обычно не останавливают, но текст лучше прочитать.

---

## Сборка

```bash
yarn build
```

Перед сборкой снова запускается проверка. Если она не прошла, сборка прервётся.

---

## Где лежит готовый сайт

После успешной сборки всё нужное для выкладки — в папке `**public/**` в корне проекта. Эту папку и нужно разместить на хостинге или отдать тем, кто публикует мини-приложение.

---

## Предпросмотр на компьютере (необязательно)

После сборки можно локально открыть собранный вариант:

```bash
yarn dev
```

Откроется адрес с портом **8080**; перед публикацией лучше сделать `yarn build`.
