## 1. Поддерживаемые счётчики

### MyTracker

- Регистрация и получение **Tracker ID**: [tracker.my.com](https://tracker.my.com)
- При `enabled: true` и непустом `trackerId` подключается скрипт аналитики; цели уходят как события конверсии (`reachGoal`)

### Яндекс.Метрика

- Создание счётчика: [metrika.yandex.ru](https://metrika.yandex.ru)

---

## 2. Конфигурация

В `static/region.json`, раздел `analytics`:

```json
{
  "analytics": {
    "mytracker": {
      "enabled": true,
      "trackerId": "3755075"
    },
    "yandexMetrika": {
      "enabled": true,
      "counterId": "85500886"
    }
  }
}
```

- `enabled: true` и заполненный идентификатор — счётчик активен
- `enabled: false` или пустой идентификатор — счётчик не подключается

## 3. События-цели

Приложение отправляет следующие цели во все включённые счётчики.

### `app_open`

|           |                                                        |
| --------- | ------------------------------------------------------ |
| **Когда** | После старта приложения (один раз за сессию загрузки) |
| **Параметры** | Нет                                                |

### `search_query`

|           |                                                                                      |
| --------- | ------------------------------------------------------------------------------------ |
| **Когда** | Пользователь ввёл поисковый запрос (от 2 символов, с небольшой задержкой — debounce) |
| **Параметры** | `query` — текст запроса после обрезки пробелов                          |

### `search_empty`

|           |                                                                          |
| --------- | ------------------------------------------------------------------------ |
| **Когда** | Поиск выполнен, результатов нет; уходит сразу после `search_query` при пустой выдаче |
| **Параметры** | `query` — текст запроса                                        |

### `service_click`

|           |                                                                                                                                   |
| --------- | --------------------------------------------------------------------------------------------------------------------------------- |
| **Когда** | Нажатие на карточку сервиса в списке (категория или «Все сервисы»)                                                                  |
| **Параметры** | `serviceId`, `serviceName`, `categoryId` (на экране всех категорий может не быть)                                              |

### `service_open`

|           |                                                                                          |
| --------- | ---------------------------------------------------------------------------------------- |
| **Когда** | Нажатие кнопки «Открыть» на странице сервиса                                              |
| **Параметры** | `serviceId`, `serviceType` (`chatbot`, `site`, `miniApp`, `form`, `channel`), `link`   |

### `category_click`

|           |                                                           |
| --------- | --------------------------------------------------------- |
| **Когда** | Выбор категории на главной (плитка или чип-фильтр)         |
| **Параметры** | `categoryId`, `categoryName`                          |

### `channel_click`

|           |                                                      |
| --------- | ---------------------------------------------------- |
| **Когда** | Нажатие на канал в карусели на главной                 |
| **Параметры** | `channelName`, `link`                            |

### `support_click`

|           |                                                                    |
| --------- | ------------------------------------------------------------------ |
| **Когда** | Нажатие на элемент блока поддержки на главной                       |
| **Параметры** | `supportType` (`chatbot`, `operator`, `techSupport`), `link`   |

Персональные данные пользователей в эти события не передаются.

## 4. Виртуальные просмотры страниц

Дополнительно при открытии экранов отправляются просмотры страниц (для отчётов по URL внутри одного SPA).

| Страница    | Маршрут                 | Просмотр |
| ----------- | ----------------------- | :------: |
| Главная     | `/`                     |    Да    |
| Категория   | `/category/:categoryId` |    Да    |
| Все сервисы | `/categories`           |    Да    |
| Сервис      | `/service/:serviceId`   |    Да    |
| Ошибка      | `/error`                |   Нет    |

## 5. Настройка целей в кабинетах

### MyTracker

Отдельная ручная настройка каждой цели в интерфейсе не требуется: события `reachGoal` попадают в отчёты по целям/конверсиям в соответствии с документацией MyTracker.

### Яндекс.Метрика

Чтобы видеть события в отчётах, создайте цели типа **JavaScript-событие** и укажите идентификатор как имя из первого столбца:

| Название для отчёта        | Идентификатор цели |
| -------------------------- | ------------------ |
| Открытие приложения        | `app_open`         |
| Поисковый запрос           | `search_query`     |
| Пустой результат поиска    | `search_empty`     |
| Клик по сервису            | `service_click`    |
| Открытие сервиса (кнопка)  | `service_open`     |
| Клик по категории          | `category_click`   |
| Клик по каналу             | `channel_click`    |
| Клик по поддержке          | `support_click`    |

Либо включите **автоцели** в настройках счётчика — часть действий может подхватываться автоматически (уточняйте по документации Яндекса для вашей версии счётчика).

---

Перед уходом со страницы по внешним ссылкам (кнопка «Открыть», поддержка) отправка цели слегка опережает открытие ссылки, чтобы счётчик успел зафиксировать событие.
